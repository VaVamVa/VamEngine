# DX3D 프로젝트 컨텍스트

## 프로젝트 개요
- **목적**: DirectX12 및 Vulkan을 이용한 크로스 플랫폼 대응 엔진 개발
- **언어**: 한국어
- **개발 방식**: 직접 구현보다는 가이드 및 개념 위주의 학습

## 작업 규칙

### 지침

- Interface 기반 C++ 3D 렌더링 엔진 개발을 돕습니다.

#### 환경
- C++20, HLSL, Visual Studio 2022, CMake
- Windows SDK / DirectX 12 최신 버전

#### 명명 규칙
- 모든 명칭은 기능을 잘 나타낼 수 있게
- 변수는 camelCase
- 함수 및 클래스/구조체는 PascalCase
    - I~~           순수 가상 클래스/구조체
    - Base~~        보편적 가상 클래스/구조체 (공통 구현 포함)
    - Impl_~~()     래퍼가 별도로 존재하는 함수(NVI 훅, Bridge 등)의 실제 구현 함수 이름

#### 코드 규칙
- ComPtr 필수 사용, RAII 원칙 준수
- HRESULT 반드시 검사, 에러 시 예외 처리
- 커맨드 리스트/큐는 목적별로 분리 (Direct/Copy/Compute)
- 리소스 상태 전환(Barrier) 명시적으로 표현


#### 응답 방식
- 코드는 컴파일 가능한 수준으로 완성도 있게 작성
- 성능 병목 가능성이 있는 부분은 주석으로 경고
- GPU 디버깅 팁(PIX, D3D12 Debug Layer) 함께 제공
- 렌더링 이론(수식 포함)과 구현을 함께 설명

**중요**:

- 목차 항목은 `[간략한 주제 설명](#qN-질문-제목)` 형태로 작성
- 마크다운 앵커는 제목의 소문자 변환 + 공백을 하이픈으로 변환하여 자동 생성

### 작업 방식
1. 사용자가 DirectX12 관련 개념/가이드 질문을 함
2. 답변 제공
3. 해당 날짜의 마크다운 파일에 Q&A 형식으로 기록
4. 같은 날짜 내 추가 질문은 같은 파일에 순차적으로 추가
5. 날짜가 바뀌면 새로운 파일 생성

### 주의사항
- 코드 직접 구현 요청이 아닌 개념 설명 위주
- 답변은 명확하고 구조화된 형태로 제공
- DirectX12 API, 렌더링 파이프라인, 최적화, 멀티 스레딩 등 엔진 개발에 필요한 모든 주제 포함
